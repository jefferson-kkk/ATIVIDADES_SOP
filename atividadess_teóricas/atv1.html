atidade 1

Um sistema operacional de código aberto permite que qualquer pessoa acesse, modifique e distribua seu código. Suas vantagens são a personalização e maior transparência. Como desvantagens, pode ser mais difícil de usar e ter pouco suporte técnico. Exemplos: Linux, FreeBSD e Android (AOSP). Já o sistema de código fechado tem o código protegido pela empresa criadora, sem possibilidade de alterações pelo usuário. Suas vantagens são o suporte técnico especializado e a facilidade de uso. As desvantagens são a falta de liberdade e o custo. Exemplos: Windows, macOS e iOS.

atividade 2

istemas operacionais de código aberto têm o código liberado para uso, modificação e distribuição. São personalizáveis e transparentes, mas podem ser difíceis de usar e ter pouco suporte. Exemplos: Linux, FreeBSD e Android (AOSP). Sistemas de código fechado têm o código protegido e controlado por empresas. São fáceis de usar e têm suporte técnico, mas não podem ser modificados e geralmente são pagos. Exemplos: Windows, macOS e iOS.

atividade 3

Existem dois tipos de interface em sistemas operacionais: o modo texto (linha de comando) e a interface gráfica (GUI). A interface gráfica é mais fácil de usar, ideal para tarefas do dia a dia, como navegar na internet, abrir programas e organizar arquivos. Ela é mais acessível, especialmente para quem não tem muita experiência com computadores. Já o modo texto exige que o usuário digite comandos, o que pode parecer difícil no começo. Porém, ele é muito útil para quem tem mais conhecimento, pois permite realizar tarefas de forma rápida, precisa e com menos uso de memória do sistema. Por isso, é bastante usado por profissionais de tecnologia, especialmente em servidores e configurações mais avançadas. Em resumo, a interface gráfica é mais indicada para usuários comuns e tarefas simples, enquanto o modo texto é melhor para quem busca controle e eficiência. O ideal é conhecer os dois e usar cada um conforme a necessidade.

atividade 4

A estrutura de diretórios do Windows começa pela unidade **C:**, com pastas como: C:\Windows: armazena os arquivos do sistema operacional. C:\Program Files: guarda os programas instalados. C:\Users: contém os arquivos e configurações dos usuários. Já no Linux, tudo começa pela raiz /, com diretórios como: /bin: contém comandos essenciais do sistema. /etc: guarda os arquivos de configuração. /home: onde ficam os arquivos pessoais dos usuários. Em resumo, ambos os sistemas organizam os arquivos por função, mas o Windows usa letras de unidade e uma estrutura mais voltada ao usuário final, enquanto o Linux usa uma hierarquia única com foco técnico e modular

atidade 5

O gerenciamento de usuários e permissões em sistemas operacionais é o processo que controla quem pode acessar e modificar arquivos, pastas e recursos do sistema. Isso é importante para garantir a segurança e a organização, evitando que usuários não autorizados façam alterações indevidas. No Windows, que é um sistema de código fechado, as permissões são gerenciadas principalmente pelo sistema de arquivos NTFS. Ele permite definir permissões detalhadas para arquivos e pastas, como leitura, escrita e execução, podendo ser configuradas para usuários e grupos específicos. Além disso, o Windows utiliza contas de usuário com diferentes níveis de privilégio, como administrador e usuário comum, para controlar o acesso. No Linux, que é um sistema de código aberto, as permissões são baseadas em um modelo simples de três tipos de usuários: o dono do arquivo, o grupo e os outros. Cada um pode ter permissões de leitura (r), escrita (w) e execução (x), representadas por símbolos (rwx). Por exemplo, um arquivo pode ter permissão de leitura e escrita para o dono, apenas leitura para o grupo e nenhuma para os outros. Esse modelo é aplicado a arquivos e diretórios e pode ser ajustado com comandos específicos. Em resumo, o Windows oferece um sistema de permissões mais detalhado e complexo, com controle via interface gráfica e políticas de segurança avançadas, enquanto o Linux usa um modelo mais simples e direto, que pode ser facilmente configurado via linha de comando, oferecendo flexibilidade e transparência.

atividade 6

Quando o computador é ligado, ele faz um autoteste (POST) para verificar o hardware, depois carrega o firmware (BIOS ou UEFI) que prepara o sistema para iniciar. Em seguida, o carregador de boot (bootloader) é executado para carregar o sistema operacional na memória. O sistema então inicia seus processos e, por fim, mostra a interface para o usuário. O processo é parecido em sistemas de código aberto (como Linux) e fechado (como Windows), mas o Linux usa bootloaders mais flexíveis, como o GRUB, que permitem escolher sistemas diferentes, enquanto o Windows usa um bootloader próprio mais simples e focado em seu sistema.

atividade 7

a) Unidades e pastas em rede são espaços de armazenamento compartilhados em outros computadores dentro de uma rede, usados para facilitar o acesso e a troca de arquivos entre usuários. b) Compartilhamento é disponibilizar esses recursos para outros na rede, e acesso é a permissão para usar esses recursos, controlada por permissões. c) Mapeamento de unidade de rede é criar um atalho que conecta uma pasta ou unidade compartilhada, fazendo-a aparecer como uma unidade local no computador, facilitando o acesso.

atidade 8

a) Segurança Cibernética é o conjunto de práticas e tecnologias usadas para proteger computadores, redes, programas e dados contra ataques, danos ou acesso não autorizado. b) 5 tipos comuns de ameaças cibernéticas: Vírus: Programas maliciosos que se replicam e podem danificar arquivos ou sistemas. Phishing: Tentativas de enganar usuários para obter informações pessoais, como senhas, por meio de mensagens falsas. Ransomware: Malware que bloqueia o acesso a arquivos e pede um resgate para liberá-los. Spyware: Programas que espionam e coletam informações sem o conhecimento do usuário. Ataques de força bruta: Tentativas repetidas de descobrir senhas ou acessos por tentativa e erro. c) Vulnerabilidades são falhas ou pontos fracos em um sistema que podem ser explorados por atacantes para causar danos ou obter acesso não autorizado. d) Credenciais são informações usadas para autenticar a identidade de um usuário, como senhas e nomes de usuário. Proteger essas informações é essencial para evitar que pessoas não autorizadas acessem sistemas e dados. e) Engenharia Social é uma técnica que usa manipulação psicológica para enganar pessoas e obter informações confidenciais. Exemplos: Um golpista que finge ser funcionário do suporte técnico para pedir a senha do usuário. Um falso e-mail pedindo que a vítima clique em um link para “atualizar” sua conta bancária.

atividade 9

Para aumentar a segurança de um sistema operacional e proteger os dados, usuários e administradores podem adotar várias intervenções e métodos de prevenção, como: Atualizações regulares: Manter o sistema operacional e os programas sempre atualizados para corrigir falhas e vulnerabilidades. Uso de antivírus e antimalware: Instalar e manter programas de segurança que detectam e eliminam ameaças. Senhas fortes e autenticação múltipla: Criar senhas difíceis de adivinhar e, quando possível, usar autenticação em duas etapas para dificultar acessos não autorizados. Controle de permissões: Definir quem pode acessar ou modificar arquivos e programas, limitando privilégios apenas ao necessário. Backup de dados: Fazer cópias de segurança regularmente para evitar perda de informações em caso de ataque ou falha. Um firewall nativo do sistema operacional funciona como uma barreira que monitora e controla o tráfego de entrada e saída da rede, bloqueando acessos suspeitos ou não autorizados. Ele ajuda a proteger o computador contra invasões e ataques externos, filtrando as conexões que podem ou não passar, aumentando a segurança geral do sistema.

atividade 9

Cinco áreas de configuração críticas em um sistema operacional que administradores ou usuários avançados precisam conhecer são: Gerenciamento de usuários e permissões: Controlar quem pode acessar o sistema e quais ações podem executar para garantir segurança e organização. Configuração de rede: Ajustar conexões, firewalls, endereços IP e permissões de acesso para garantir comunicação segura e eficiente. Atualizações do sistema: Manter o sistema e softwares atualizados para corrigir falhas, melhorar desempenho e aumentar a segurança. Gerenciamento de armazenamento: Organizar discos, particionar unidades e monitorar espaço disponível para otimizar a performance e evitar problemas de falta de espaço. Serviços e processos de inicialização: Configurar quais programas e serviços iniciam junto com o sistema para melhorar a velocidade de boot e liberar recursos para tarefas importantes.
